N4j

Neo4j Cypher MCP Server

Read-only graph access over the Model Context Protocol. Hosted on Databricks Apps.
Running Transport Streamable HTTP Endpoint POST /mcp/mcp Mode read-only Backend Neo4j Aura

This page is not the API

You're looking at a server, not a website. The browser-visible / exists so demos don't see a bare 404. The actual protocol surface is POST /mcp/mcp — the Databricks Apps MCP registry treats the app URL as the MCP base and appends /mcp to construct the protocol endpoint, so the canonical URL ends in /mcp/mcp. Consumed by MCP-aware clients: the Databricks Supervisor API, Claude Desktop, Cursor, custom agents, etc.

Tools exposed

get_neo4j_schema(sample_size: int = 100)
Inspect labels, relationship types, property keys, and the most common (a)-[r]->(b) patterns. Uses apoc.meta.schema when available, otherwise falls back to native db.labels() / db.relationshipTypes() / db.propertyKeys().
read_neo4j_cypher(query: str, params?: dict, row_limit: int = 50)
Execute one read-only Cypher statement. Mutating clauses (CREATE/MERGE/DELETE/SET/REMOVE/DROP/...) are rejected at the server. Returns JSON-serialised rows including nodes, relationships, paths, and temporals.

Try it (terminal)

TOKEN=$(databricks auth token --profile <profile> | jq -r .access_token)
URL=https://<this-app>/mcp/mcp

# MCP initialize -> SSE response with server capabilities
curl -sN -H "Authorization: Bearer $TOKEN" \
  -H "Content-Type: application/json" \
  -H "Accept: application/json,text/event-stream" \
  "$URL" -d '{
    "jsonrpc":"2.0","id":1,"method":"initialize",
    "params":{"protocolVersion":"2025-03-26","capabilities":{},
              "clientInfo":{"name":"smoke","version":"0.1"}}}'

Consume from a Databricks agent

# Supervisor API hosted-tool declaration in agent.py
TOOLS = [
    {"type": "app",
     "app": {"name": "mcp-neo4j-cypher",
             "description": "Read-only Cypher and Neo4j schema introspection."}},
]

# databricks.yml grant on the agent app
resources:
  - name: mcp_neo4j_cypher
    app: { name: mcp-neo4j-cypher, permission: CAN_USE }

No manual registration needed — any Databricks App with an mcp- name prefix is auto-listed in Agents > MCPs.
